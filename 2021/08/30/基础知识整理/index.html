闭包 （旧版模块化开发）（函数嵌套函数）

1.犀牛书：函数变量可以保存在函数作用域内。–函数外部不能访问函数内部变量
2.高级程序设计：有权访问另一个函数作用域中变量的函数。–函数嵌套函数，内函数为闭包，可访问外函数变量。
3.你不知道的js:当函数可以记住并访问所在的词法作用域时，就产生了闭包，即使函数在当前词法作用域之外执行。—函数嵌套函数，只要内函数有输出，就可以在全局调用内函数
4.mdn:一个函数和它周围状态的引用捆绑在一起的组合
5.总结:当函数的执行，导致内函数被定义就是闭包
闭包使用原因：防抖节流用闭包，原因是内函数能用外部变量，而外部变量不影响全局变量。

使用优点：1.避免变量全局污染
让变量常驻内存不会被【垃圾回收机制所回收】
2.私有函数和变量
缺点：外函数中嵌套内函数，外函数作用域中变量因为被内函数引用未被销毁，所以一直存储在内存中，当变量过多或者嵌套层级过多时有内存泄漏的风险
解决办法:
1.当内部变量不用时，把外部变量置为null
2.减少嵌套层级，最外层不创建变量。

    递归（自己调用自己）

    根据内存管理机制，递归会瞬间开启并消除内存，如果过大的话，会导致计算机崩溃

1.首先找到临界值，即无需计算，获得的值。
2.找这一次和上一次的关系
3.假设当前函数已经可以使用，调用自身计算上一次

实例：计算1~n之和
function sum(n){
if(n＝＝1){return 1}
return sum(n-1)+n
}

斐波那契数列
n＝(n-1)+(n-3)

    事件轮询(event loop)执行方式

    分为调用栈–消息队列—回调队列
    js是单线程的，页面执行脚本时会在调用栈执行任务，当遇到setTimeout等异步任务时会挂在消息队列末端等待，此时同步进程继续执行，等异步任务时间结束后会到回调队列中，通过事件轮询的方式，找到回调队列的任务放到调用栈中执行。所以事件轮询的作用是不断查看回调队列中是否有未执行任务，有就放到调用栈执行。
